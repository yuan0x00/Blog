一、Kotlin 基础语法（必考） ​

1. Kotlin 与 Java 的主要区别？ ​

✅ 参考答案： ​

特性KotlinJava
空安全编译期检查，String? 表示可空无，运行时 NPE

💡 加分点：Kotlin 编译为 JVM 字节码，100% 与 Java 互操作。

2. val 和 var 的区别？ ​

✅ 参考答案： ​

  • val：只读引用（类似 Java final），初始化后不能重新赋值。
    kotlin
    val name = "Alice" // OK
name = "Bob"       // 编译错误

  • var：可变引用，可重新赋值。
    kotlin

⚠️ 注意：val 修饰的对象内部状态仍可变（如 val list = mutableListOf(1,2) 可 list.add(3)）。

3. 空安全机制如何工作？ ​

✅ 参考答案： ​

Kotlin 通过 类型系统区分可空与非空类型：

  • 非空类型：String → 不能为 null
  • 可空类型：String? → 可为 null

安全调用操作符 ?. ​

kotlin
val length = str?.length // 若 str 为 null，结果为 null
1

Elvis 操作符 ?: ​

kotlin
val name = user?.name ?: "Unknown" // null 时提供默认值
1

非空断言 !! ​

kotlin
val length = str!!.length // 若 str 为 null，抛出 KotlinNullPointerException
1

安全转换 as? ​

kotlin
val str = obj as? String // 转换失败返回 null
1

💡 核心思想：编译期杜绝 NPE，将运行时错误转为编译错误。

三、协程（Coroutine）—— 高频重点！ ​

7. 协程是什么？相比线程/AsyncTask/RxJava 优势？ ​

✅ 参考答案： ​

  • 协程：轻量级 并发设计模式（非线程），由 Kotlin 官方库实现。
  • 核心优势：
    • 轻量：1 个线程可运行数千协程（线程栈 ~1MB，协程栈 ~KB）
    • 语法简洁：suspend 函数 + 结构化并发
    • 异常处理统一：CoroutineExceptionHandler
    • 避免回调地狱：顺序写法处理异步

对比： ​

方案问题
Thread资源消耗大，切换开销高
AsyncTask已废弃，内存泄漏风险
RxJava学习曲线陡峭，背压复杂
协程官方推荐，与 Kotlin 深度集成

💡 Android 官方推荐：协程是 Android 异步编程的首选方案。

8. suspend 函数原理？为什么不能在主线程直接调用？ ​

✅ 参考答案： ​

  • suspend 函数：可挂起和恢复的函数，只能在协程或其他 suspend 函数中调用。
  • 原理：
    • 编译器将 suspend 函数转换为 状态机（带 Continuation 回调）
    • 挂起时保存状态，恢复时从断点继续执行
  • 为什么不能在主线程直接调用？
    • suspend 函数本身不切换线程！
    • 若在主线程调用耗时 suspend 函数（如 delay(5000)），仍会阻塞主线程。
    • 正确做法：用 withContext(Dispatchers.IO) 切换到后台线程。

示例： ​

kotlin
// ❌ 错误：在主线程执行耗时操作
suspend fun fetchData() {
    val data = api.getData() // 网络请求（应放 IO 线程）
}

// ✅ 正确
suspend fun fetchData() = withContext(Dispatchers.IO) {
    api.getData()
}

9. 协程作用域（Scope）与结构化并发 ​

✅ 参考答案： ​

  • 结构化并发：协程必须在 作用域（Scope） 内启动，父协程取消时自动取消所有子协程。
  • 常用 Scope：
    • viewModelScope（ViewModel 中）：Activity 销毁时自动取消
    • lifecycleScope（Activity/Fragment 中）：生命周期结束时取消
    • CoroutineScope：自定义作用域（需手动管理生命周期）

示例： ​

kotlin
// ViewModel 中
fun loadData() {
    viewModelScope.launch {
        val data = repository.fetchData()
        _uiState.value = data
    }
}
// Activity 销毁时，viewModelScope 自动取消协程

五、性能与进阶（高级考点） ​

12. Kotlin 编译后的字节码有什么特点？如何优化？ ​

✅ 参考答案： ​

  • 特点：
    • 扩展函数 → 静态方法
    • 数据类 → 自动生成 equals/hashCode/toString
    • 协程 → 状态机类（带 Continuation）
  • 优化建议：
    • 避免在循环中创建 lambda（用 inline 函数）
    • 用 @JvmStatic/@JvmField 减少包装
    • 启用 R8 代码压缩（自动优化 Kotlin 字节码）

🔍 工具：Android Studio → Tools > Kotlin > Show Kotlin Bytecode

13. inline、noinline、crossinline 的作用？ ​

✅ 参考答案： ​

  • inline：内联函数，编译时将函数体复制到调用处，避免 lambda 对象创建。
    kotlin
    inline fun <T> lock(lock: Lock, body: () -> T): T {
    lock.lock()
    try { return body() } finally { lock.unlock() }
}

  • noinline：标记 lambda 不内联（如需作为参数传递）
    kotlin
    inline fun foo(a: () -> Unit, noinline b: () -> Unit) {
    a() // 内联
    someFunction(b) // b 不能内联（需对象引用）
}

  • crossinline：禁止 lambda 中使用 非局部返回（如 return@foo）
    kotlin
    inline fun runAfter(delay: Long, crossinline block: () -> Unit) {
    thread { 
        Thread.sleep(delay)
        block() // 若 block 有 return，会退出 thread 而非 runAfter

1. 实现一个安全的 SharedPreferences 委托 ​

kotlin
object SharedPreferencesDelegate {
    fun <T> SharedPreferences.property(
        key: String,
        defaultValue: T
    ): ReadWriteProperty<Any?, T> = when (defaultValue) {
        is String -> StringPreference(this, key, defaultValue)
        is Boolean -> BooleanPreference(this, key, defaultValue)
        else -> throw IllegalArgumentException("Unsupported type")
    }
}

private class StringPreference(
    private val prefs: SharedPreferences,
    private val key: String,
    private val default: String
) : ReadWriteProperty<Any?, String> {
    override fun getValue(thisRef: Any?, property: KProperty<*>): String =
        prefs.getString(key, default) ?: default

    override fun setValue(thisRef: Any?, property: KProperty<*>, value: String) {
        prefs.edit().putString(key, value).apply()
    }
}

// 使用
class Settings(context: Context) {
    private val prefs = context.getSharedPreferences("app", Context.MODE_PRIVATE)
    var username: String by prefs.property("username", "")
}

2. 协程封装网络请求 ​

kotlin
class UserRepository {
    suspend fun getUser(id: Int): Result<User> = withContext(Dispatchers.IO) {
        try {
            val response = api.getUser(id)
            Result.success(response)
        } catch (e: Exception) {
            Result.failure(e)
        }
    }
}

// ViewModel 中调用
fun loadUser(id: Int) {
    viewModelScope.launch {
        when (val result = repository.getUser(id)) {
            is Result.Success -> _user.value = result.data
            is Result.Failure -> _error.value = result.exception
        }
    }
}
